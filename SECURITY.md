# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AI-IDLE, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please email: **[ai-idle@outlook.com](mailto:ai-idle@outlook.com)**

We will acknowledge receipt within 48 hours and work with you to understand and address the issue.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.2.x   | :white_check_mark: |
| 1.1.x   | :white_check_mark: |
| 1.0.x   | :x:                |

## Scope

This security policy covers the AI-IDLE platform, including:
- Backend API
- Frontend application
- Authentication and authorization
- Data handling and storage
- Network communication

## Response Timeline

| Step | Timeline |
|------|----------|
| Acknowledgment | 48 hours |
| Initial assessment | 5 business days |
| Fix development | Depends on severity |
| Public disclosure | After fix is deployed |

Thank you for helping keep AI-IDLE secure.
